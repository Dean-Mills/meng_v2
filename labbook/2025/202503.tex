You can also use align

\begin{align}
	\m{QX} \m{a}                             & = \m{Qy} \notag                                                        \\
	\therefore \quad (\m{QX})^T \m{QX} \m{a} & = (\m{QX})^T \m{Qy} \notag                                             \\
	\therefore \quad \m{a}                   & = \left( (\m{QX})^T \m{QX} \right)^{-1} (\m{QX})^T \m{Qy} \notag       \\
	                                         & = \left( \mt{X} \mt{Q} \m{QX} \right)^{-1} \mt{X} \mt{Q} \m{Qy} \notag \\
	                                         & = \left( \mt{X} \m{W} \m{X} \right)^{-1} \mt{X} \m{W} \m{y}
	\label{eqn:weightedleastsquares}
\end{align}

\section[2025/03/28]{Friday , 28 March 2025}
\label{sec:20250328}

Investigating a new idea

\begin{compactitem}
	\item Use pytorch to load data
	\item use ResNet or HRNet to create heatmaps (probability distributions for each joint)
	\item Use depth estimator to get depth and create a 3D probability "sphere"
	\item This can then be used in GAT or Transformer
\end{compactitem}

The idea is that the 3D sphere will have additional features like entropy (a measure of spread),
spatial coordinates, confidence scores from the heatmap etc

The graph is easier to visualize because each node is a heatmap (sphere) and has a set of associated
features. These nodes are then connected to each other. Transformers will have to do more research
but the idea is that transformers view each joint detectionn (or even each person) as a token and uses
self-attention to reason about their relationships

Some example of state of the art monocular depth estimators

\begin{compactitem}
	\item MiDaS
	\item ZoeDepth (2023)
	\item PatchFusion (2023)
	\item Marigold (CVPR 2024)
\end{compactitem}
